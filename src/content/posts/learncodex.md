顺带提一下，作为一款开源的，方便一键切换API的工具，ccswitch巧妙地解决了官方API，中转站等等需要频繁手动切换的痛点，而且提供了图形化界面可以更方便的管理，这里特别推荐，开源链接为[CC-Switch](https://github.com/farion1231/cc-switch)

其实我们可以使用这个工具辅助配置codex和claude code，但是不用也完全没关系的（

因为这次主要是教你如何改配置，ccswitch主要就是在此基础上做了更方便的管理，所以我先从最底层的手动改配置文件讲起来

因为是面向想初步接触的人群，那还是需要简单介绍一下codex和claude code

可能会有一些常见的问题，比如，平常在deepseek chatgpt等等网页问个问题，提个要求帮我写代码，和这个又有什么不一样呢？明明都是让AI给我干活了

这两个东西，本质上是是一个 AI 编程代理（AI Coding Agent），目标不是帮你补全代码，而是直接参与并完成整个开发任务——写代码、修 Bug、跑测试、提pr等等...真正的实现全自动化

我们平常使用的网页端 本质上是我们把自己的需求提给AI 然后为我们生成代码 我们本地进行调试运行 然后继续复制粘贴 喂给AI 

这种其实就是一个语言模型 只能做到推理代码，碰不到那些报错信息，每次工具调用都需要我们人来手动把结果粘贴过去

The Agent Loop(智能体循环)  一个工具 + 一个循环 = 一个智能体，显而易见，从我这个说法可以看出 这里循环就是我们人类自己本身

但是claude code这种工具与这些网页端AI不同之处在哪呢？就在于可以自行循环，自行检索收集结果，作为user消息追加，取代了人类复制粘贴的过程，这就是外层编排 + 内层循环 + 模型流式适配 三层，跑通闭环，实现真正的智能agent

所以这里就很容易理解了，这种模式明显更适合我们日常搞搞开发之类的使用，更偏向实用性，加入各种实用的skills之后简直完爆网页端....插句题外话，其实我建议计算机专业的大学生都要来尝试一下这种agent工作的感觉，而不是一味地提要求，复制粘贴代码，越早感受一下vibe coding的感觉越好，因为未来肯定还是AI的天下，发展进步实在是太快了（
